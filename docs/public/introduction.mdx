# manymems

**Open-source memory and code-provenance dashboard**

manymems captures Claude Code sessions into a local SQLite index with FTS5 search, then surfaces them through a team-oriented dashboard. Each session's observations — file reads, writes, tool calls — get linked to Git commits and previous context, giving teams visibility into how AI-assisted work shapes their codebase.

## Quick Start

Clone the repository and run the development worker:

```bash
git clone https://github.com/chicago-joe/manymems.git
cd manymems
npm install
npm run build
```

Start the worker on port 37778:

```bash
export PATH="$PATH:/home/chicagojoe/.bun/bin"
CLAUDE_MEM_DATA_DIR=~/.claude-mem \
CLAUDE_MEM_WORKER_PORT=37778 \
CLAUDE_MEM_WORKER_HOST=127.0.0.1 \
CLAUDE_MEM_CHROMA_ENABLED=false \
CLAUDE_MEM_LOG_LEVEL=warn \
  bun plugin/scripts/worker-service.cjs --daemon
```

Open http://localhost:37778 in your browser. The dashboard captures all Claude Code sessions automatically via hooks.

## Key Features

- **Session Capture** - Automatically records file reads, writes, and all tool calls via Claude Code hooks
- **Dashboard UI** - Prefect Cloud-style dark interface with observation timeline and drill-down
- **Local SQLite Index** - FTS5 full-text search across all captured sessions
- **Git Attribution** - Links observations to commits with author and timestamp
- **API Explorer** - Browse the worker HTTP API directly from the dashboard
- **MCP Search Tools** - Query sessions and observations from any Claude Code session
- **Privacy by Default** - No network calls; all data stays local unless you sync it

## How It Works

```
┌───────────────────────────────────────────────────────────┐
│ Claude Code Hook → Capture tool usage & prompts          │
└───────────────────────────────────────────────────────────┘
                          ↓
┌───────────────────────────────────────────────────────────┐
│ Local Worker (port 37778) → Index into SQLite + FTS5      │
└───────────────────────────────────────────────────────────┘
                          ↓
┌───────────────────────────────────────────────────────────┐
│ Dashboard Web UI → Search, filter, drill into sessions    │
└───────────────────────────────────────────────────────────┘
                          ↓
┌───────────────────────────────────────────────────────────┐
│ MCP Tools → Query from Claude Code for future context     │
└───────────────────────────────────────────────────────────┘
```

**Core Components:**
1. **Claude Code Hooks** - Capture observations at SessionStart, PostToolUse, SessionEnd
2. **Worker Service** - Bun-based Express HTTP API on port 37778
3. **SQLite Database** - Stores sessions, observations, and metadata with FTS5 indexing
4. **Web Viewer** - React dashboard with real-time SSE updates
5. **MCP Search Tools** - Natural language query interface for Claude Code

See [Architecture Overview](architecture/overview) for details.

## System Requirements

- **Node.js**: 20.0.0 or higher
- **Bun** ≥ 1.0
- **Git**: for commit attribution
- **SQLite 3** — bundled via `bun:sqlite`
